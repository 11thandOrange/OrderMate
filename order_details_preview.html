#45 Clover Tags vs OrderMate Tags - Logic Explained

Header shows Clover Tags (order.state + order.paymentState):

• orderPlacedStatusValue → order.state: OPEN or CLOSED (LOCKED)
• paymentStatusBadge → order.paymentState: PAID, UNPAID, PARTIALLY PAID, PARTIALLY REFUNDED, REFUNDED

Tags Row shows Custom OrderMate Tags (from order.note):

Custom tags extracted from order notes like: Category: Birthday • Type: Pickup • Tag: Rush

Order State (order.state):

OPEN - Order is active
CLOSED - Order is locked/finalized

Payment State (order.paymentState):

PAID - Fully paid
UNPAID - No payment (NOT_PAID)
PARTIALLY PAID
PARTIALLY REFUNDED
REFUNDED

Example Header Combinations:

OPEN PAID
OPEN UNPAID
OPEN PARTIALLY PAID
CLOSED PAID
CLOSED REFUNDED

OrderMate Custom Tags (in Tags Row):

Birthday Wedding Pickup Delivery Rush VIP
#46 Order Notes Data - Description Row
