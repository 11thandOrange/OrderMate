# Autonomous Dev Pipeline — {{REPO_NAME}}

End-to-end automation: GitHub Issue labelled `ready-to-implement` →
plan → implement → test → build → draft PR → review → CI green →
mark ready → WhatsApp notification.

## Pipeline

```
Issue labelled "ready-to-implement"
        ↓
  ticket-planner       reads issue, maps to codebase, posts plan via ticket-manager
        ↓
   implementer         creates branch, writes code ({{DOMAIN_A}})
        ↓
 {{SECONDARY_IMPLEMENTER_NAME}}   if issue touches {{DOMAIN_B}} (plan determines this)
        ↓
      tester           writes missing unit tests, runs all suites
        ↓
  build-check          {{BUILD_CMD}}
        ↓
  ticket-manager       gh pr create (draft)                        [USER LEVEL]
        ↓
   pr-reviewer         self-review, inline comments (max 2 iterations)
        ↓
   ci-monitor          polls gh pr checks until green — max 3 retries  [USER LEVEL]
        ↓
 mark-pr-ready         removes draft status, triggers smoke CI          [USER LEVEL]
        ↓
 whatsapp-notifier     ✅ PR #N ready / ❌ pipeline failed              [USER LEVEL]
```

## Register the Automation

Run the registration script generated by `automation-builder`:

```bash
bash /tmp/register-automations-{{REPO_SLUG}}.sh
```

Or register manually:

```bash
curl -X POST "https://app.all-hands.dev/api/automation/v1/preset/prompt" \
  -H "Authorization: Bearer ${OPENHANDS_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "{{REPO_NAME}} — Autonomous Dev Pipeline",
    "prompt": "<rendered by automation-builder>",
    "trigger": {
      "type": "event",
      "source": "github",
      "on": "issues.labeled",
      "filter": "event.label.name == '\''ready-to-implement'\'' && glob(repository.full_name, '\''{{REPO_URL}}'\'')"
    },
    "timeout": 3600,
    "repos": [{"url": "https://github.com/{{REPO_URL}}", "ref": "main"}]
  }'
```

## Trigger the Pipeline

```bash
gh label create "ready-to-implement" --repo {{REPO_URL}} \
  --color "0075ca" --description "Queued for autonomous implementation"

gh issue edit <ISSUE_NUMBER> --repo {{REPO_URL}} --add-label "ready-to-implement"
```

## What the Pipeline Will Never Do

- Merge to `main`
- Run production deploys
- Modify environment files
